# 参与贡献 Timet

感谢你帮助 Timet 变得更完整、更可信。

## 提交 PR 之前

- 先搜索已有 issue，避免重复工作。
- 尽量提交小而聚焦的 PR，而不是大范围重构。
- 如果改了 UI，最好附上截图或录屏。
- 如果修的是运行时问题，请写清设备型号、系统版本和复现步骤。

## 本地开发

```bash
npm install
npm run knowledge:build
npm test
npm run build
```

如果你涉及 Dart 后端模块或原生壳层，再补跑：

```bash
dart test
npm run mobile:build
```

## Pull Request 检查清单

- 保持产品核心仍然是“穿越策略问答”，不要滑回通用应急工具。
- 不要引入任何“假 AI fallback”。
- 保持离线优先，不能把关键路径改成依赖网络。
- 如果修改了 prompt、路线命中、检索、会话状态或 UI 流程，请同步补充或更新测试。
- 所有用户可见文案都要清楚、短促、可执行。

## 知识包贡献

- 优先使用可再分发、可追溯、历史语境清楚的资料和整理结果。
- 每个来源都要确认版权与再分发条件。
- 新来源请通过 manifest 和构建脚本接入，不要手改生成产物。
- 路线内容要强调 `先走三步`、`收益`、`禁忌`、`掩护话术`，而不是只堆背景知识。

## 涉及安全的改动

如果你的改动涉及模型打包、原生桥接、权限边界或信任提示，请在 PR 里写更详细的背景和验证方式，方便重点审查。
